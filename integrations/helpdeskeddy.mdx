Вы можете самостоятельно настроить интеграцию с HelpDeskEddy и начать обрабатывать обращения пользователей с использованием нашего бота.

<Steps>
   <Step title="Настройте интеграцию">
    На [портале](https://app.wikibot.pro) перейдите в раздел **Подключения** своего бота и включите интеграцию HelpDeskEddy.
    Заполните API секцию:
      - **Домен** – Ваш домен HelpDeskEddy.
      - **Почта** – почта сотрудника в HelpDeskEddy.
      - **API ключ** – создаётся по пути: Контакты → выбор сотрудника → API key 
      <br/>(на странице `https://<ваш домен>.helpdeskeddy.com/ru/contacts/`).

    Если все три параметра заполнены верно, появится возможность выбрать агента:
      - **Агент** (опционально) – сотрудник, на которого будут переводиться обращения, в случае если бот не нашел ответа, либо пользователь позвал оператора.

      - Также придумайте название метки, на которую будет реагировать Wikibot, например `wikibot`
    <Note>
      Можно выбрать пункт **Посылать ответ как заметку**, тогда ответы бота будут видеть только агенты, пользователи видеть их не будут.
    </Note>

    Нажмите **Добавить**, чтобы сохранить интеграцию.

  </Step>
  <Step title="Скопируйте Webhook">
    В настройках интеграции скопируйте в буфер обмена Webhook URL Вашего бота.    
    <img
      src="/images/integrations-hde-webhook.jpg"
      alt="hde webhook"
    />
  </Step>
   <Step title="Настройте правило для добавления метки Wikibot">
    Wikibot будет обрабатывать только те заявки, которые содержат метку. Если метка сбрасывается, Wikibot перестает реагировать на заявку.

    Для создания правила, перейдите по пути: «Управление» → «Диспетчер» и нажмите кнопку «Добавить правило».

    Нужно задать обязательное условие:
      - Новая заявка.

    Также задайте действие:
      - Добавить метки - используйте название, которое вы придумали в пункте 1.

    <img
      src="/images/integrations-hde-label.jpg"
      alt="hde webhook"
    />
    </Step> 

  <Step title="Настройте правило для новых заявок">

    На странице правил, добавьте новое правило для входящих заявок.

    Нужно задать обязательное условие:
      - Новая заявка.

    Также опционально можно задать дополнительное условие:
      - Автор последнего ответа - Равно - Клиент

    Действие:
      - Отправить вебхук, тип запроса POST, формат JSON, без авторизации.
      - Укажите URL, который вы скопировали в пункте 2.
      - Укажите тело запроса:
      ```
    {
        "id": {ticket_id},
        "message": "{answer_last}",
        "tags": "{ticket_tags}",
        "userId": {user_id},
        "files": "{last_answer_attachments_links}"
     }
      ```
    <img src="/images/integrations-hde-new.jpg" alt="new request hde" />
    
  </Step>
   <Step title="Настройте правило на изменение заявки">

    На странице правил, добавьте новое правило для изменения заявок. Правило будет срабатывать, когда в заявке появляются новые ответы.

    Нужно задать обязательное условие:
      - Новый ответ в завке.
    
    Также нужно задать два дополнительных условия:
      - Автор последнего ответа - Равно - Клиент
      - Метки заявки - Содержит - `wikibot` (название метки, которое вы придумали в пункте 1)

    Действие:
      - Отправить вебхук, тип запроса POST, формат JSON, без авторизации.
      - Укажите URL, который вы скопировали в пункте 2.
      - Укажите тело запроса:
      ```
    {
        "id": {ticket_id},
        "message": "{answer_last}",
        "tags": "{ticket_tags}",
        "userId": {user_id},
        "files": "{last_answer_attachments_links}"
     }
      ```
    <img src="/images/integrations-hde-edit.jpg" alt="new request hde" />
  </Step>
 
  <Step title="Проверьте работу бота">
    Ваш чат-бот для обработки обращений готов! Проверьте его работу в любом из каналов HelpDeskEddy.
  </Step>
</Steps>

Если у вас остались вопросы по интеграции, [напишите нам](https://t.me/use_wikibot).

<img src="/images/integrations-usedesk.jpg" alt="usedesk integration" />

Вы можете самостоятельно настроить интеграцию с Юздеск и начать обрабатывать обращения пользователей с использованием нашего бота.

 В настройке интеграции также поможет [гайд от Юздеск](https://docs.usedesk.ru/article/58413).
<Info>
  По умолчанию Wikibot отвечает только на обращения, для которых **не назначен**
  конкретный **агент**, либо назначена группа.
</Info>

<Steps>
  <Step title="Создайте API-канал в Юздеске">
    Создайте [**API-канал**](https://docs.usedesk.ru/article/10167) и укажите в качестве **Redirect url**
    `https://api.wikibot.pro/webhook/usedesk/botId`, где `botId` — это
    идентификатор вашего бота. Вы можете скопировать его на странице
    [Обзор](/lk/overview), либо запросить у [нас](https://t.me/wikibot_support).
    <img src="/images/usedesk-create-channel.jpg" alt="usedesk create channel" />
  </Step>
  <Step title="Параметры интеграции Юздеск">
    Установите галочки **JSON-запрос** и **UTF-8** - бот принимает данные только в формате JSON и в кодировке UTF-8, поэтому необходимо отметить эти опции для корректной работы интеграции.
    <img src="/images/usedesk-json-checkbox.jpg" alt="usedesk json checkbox" />
    Сохраните канал.
  </Step>
  <Step title="Задайте параметры канала">
    После сохранения в настройках канала появятся два заполненных поля: **Id приложения** и **Секретный ключ**.
    <img src="/images/usedesk-api-keys.jpg" alt="usedesk api keys" />
    Скопируйте значения **Id приложения** и **Секретный ключ** и введите их в соответствующие поля мастера настройки интеграции на [портале Wikibot](/lk/integrations)
    либо передайте значения [нам](https://t.me/wikibot_support).
  </Step>
  <Step title="Задайте подписки на каналы">
    Выберите каналы, за которыми бот должен следить.
    Это может быть чат, мессенджер или почта. Можно выбрать несколько каналов.
    <img src="/images/usedesk-choose-channel.jpg" alt="usedesk choose channel" />
    И отметьте галочками, какие действия отслеживать. Например:
    <img src="/images/usedesk-activity-tracking.jpg" alt="usedesk activity tracking" />
    При такой настройке любые новые сообщения в выбранных каналах будут попадать к боту в обработку.
    <Info>Примеры работы бота при разных настройках отслеживания каналов - по [ссылке]()</Info>
  </Step>
  <Step title="Задайте агента для ответа">
    В настройках интеграции на портале в поле **ID агента** укажите ID агента, от имени которого будет отвечать
    бот.
    ID агента вы найдете в адресной строке при редактировании агента в Юздеске:
    <img src="/images/usedesk-agents-id.jpg" alt="usedesk agents id" />
    Вы можете [создать нового внешнего агента](https://docs.usedesk.ru/article/24) - подключение нового агента может потребовать дополнительной оплаты, это зависит от вашего тарифа в Юздеск.
    
    Или можно выбрать агента из имеющихся.
    В этом случае при работе со статистикой по агентам стоит учесть, что указанный агент - это бот.
  </Step>
  <Step title="Задайте агента для перевода">
    Задайте ID агента, на которого бот будет переводить вопросы без ответов и вызовы
    оператора - поле **Assignee ID**.
    Для Assignee ID можно указать группу агентов, тогда бот будет
    назначать тикеты на онлайн-участников группы в случайном порядке.
    <Info>Агент, от имени которого отвечает бот, и агент-оператор могут быть одним и тем же агентом</Info>
    Пример заполенения ID агентов:
    <img src="/images/usedesk-set-agents.jpg" alt="usedesk set agents" />
  </Step>
  <Step title="Проверьте работу бота">
    Сохраните настройки интеграции.

    Ваш чат-бот готов! Проверьте работу чат-бота в любом из отслеживаемых каналов Юздеска.
    При желании разместите [чат-виджет](https://docs.usedesk.ru/article/846) с ботом на вашем сайте.
  </Step>
</Steps>

Если у вас остались вопросы по интеграции, [напишите нам](https://t.me/wikibot_support).

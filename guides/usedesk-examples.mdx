---
title: "Примеры настроек для Юздеск"
---

О настройке интеграции бота в Юздеск мы рассказали в [этой статье](/integrations/usedesk).

Обработка обращений будет зависеть от параметров отслеживания каналов, заданных в настройках API-канала.

В этой статье мы приведем некоторые рекомендации по настройке Юздеска.

### Настройка чата
Итак, ваш чат подключен к боту.

В настройках API-канала в Юздеске доступно 3 типа событий для отслеживания чатов:
- Новый чат;
- Новое сообщение;
- Возобновление чата.

<Note>
    Для чата мы рекомендуем выставить 2 галочки: "Новый чат" и "Новое сообщение"
</Note>
<img src="/images/usedesk-chat-activity.jpg" alt="usedesk chat activity" />
Так бот будет реагировать на создание чата, то есть **на первое сообщение в чате**,
 и **на все новые сообщения** до тех пор,
пока для чата не будет назначен агент или чат не будет закрыт.

Для примера, если задать отслеживание только события "Новый чат", то бот будет реагировать _только на первое сообщение в чате_, даже если это сообщение - приветствие.
То есть будет работать только при создании чата.

<Note>
    Мы рекомендуем **снять** [автоматическое распределение чатов](https://docs.usedesk.ru/article/6013)
</Note>
Так Юздеск будет оставлять чат без автоматического назначения агента, 
что и требуется для ответа бота: бот отвечает только на тикеты, для которых не назначен агент (при этом может быть назначена группа агентов).

<Note>
    Мы рекомендуем **отключить в Юздеске [реакцию на приветствие](https://docs.usedesk.ru/article/858)** 
    в чате, где работает бот
</Note>
Бот сам будет отвечать на приветствие, а при реакции на приветствие и из Юздеска пользователь получит два ответа на приветствие.


### Настройка почты
Для отслеживания почты в настройках API-канала в Юздеске доступно 5 типов событий:
- Новый запрос;
- Первое сообщение;
- Новое сообщение;
- Запрос изменен;
- Новая оценка.

<Note>
    Для почты мы рекомендуем выставить галочки: "Новый запрос", "Первое сообщение", "Новое сообщение"
</Note>
<img src="/images/usedesk-email-activity.jpg" alt="usedesk email activity" />

В результате все сообщения, которые отправляются в отслеживаемый почтовый канал и 
не имеют назначенного агента, будут отвечены ботом.
И бот будет отвечать на запросы, где еще не назначен агент, даже если пользователь создаст
несколько последовательных обращений в одном запросе.

<Info>
    Если бот обработал запрос, затем запросу был назначен агент, а после агент был удален,
    то при повторном сообщении пользователя бот не ответит на такой запрос.
</Info>
